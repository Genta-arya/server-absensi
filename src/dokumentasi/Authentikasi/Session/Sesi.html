API Documentation - Session Check Login

Cek Sesi Pengguna

Endpoint untuk memeriksa validitas token pengguna.

POST /session

Memeriksa apakah token yang diberikan masih valid atau telah kedaluwarsa.

Request Body:


{
    "token": "string"  // Token pengguna yang akan diperiksa
}

Response:

200 OK: Jika token masih valid.


{
    "message": "success",
    "data": {
        "id": "string",  // ID pengguna
        "nim": "string",  // NIM pengguna
        "name": "string | null",  // Nama pengguna (jika ada)
        "email": "string | null",  // Email pengguna (jika ada)
        "isLogin": true,  // Status login
        "token": "string | null",  // Token (jika ada)
        "avatar": "string | null",  // URL Avatar (jika ada)
        "role": "MHS" | "DOSEN" | "ADMIN"  // Role pengguna
    }
}

400 Bad Request: Jika token tidak disertakan dalam permintaan.


{
    "message": "Mohon lengkapi token" // Jika token tidak ada
}

401 Unauthorized: Jika token tidak valid atau sudah kedaluwarsa.


{
    "message": "Token tidak valid" // Jika token tidak ditemukan di database
}
// atau
{
    "message": "Token expired. Silakan login kembali." // Jika token sudah kedaluwarsa
}

Penggunaan

Untuk memeriksa sesi login pengguna, kirimkan permintaan POST ke /session dengan body yang berisi token pengguna.
